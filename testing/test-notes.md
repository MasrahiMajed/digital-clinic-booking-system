# Testing Notes

## Objective
The objective of testing is to verify that the Digital Clinic Booking System functions correctly and allows students, doctors, and administrators to perform their intended tasks.

## Test Scenario 1 – Student Registration
User registers a new account using email and password.  
Expected result: Account is successfully created and user can log in.

## Test Scenario 2 – Student Login
User logs in with registered credentials.  
Expected result: Student dashboard loads successfully.

## Test Scenario 3 – Booking Appointment
Student selects an available appointment slot and confirms booking.  
Expected result: Appointment is saved and visible in the student’s appointments.

## Test Scenario 4 – Cancel Appointment
Student cancels a previously booked appointment.  
Expected result: Appointment status updates and slot becomes available again.

## Test Scenario 5 – Doctor Dashboard
Doctor logs in and views scheduled appointments.  
Expected result: Doctor can see all assigned sessions.

## Test Scenario 6 – Admin Management
Admin views and manages student appointments.  
Expected result: Admin can update or cancel appointments.
